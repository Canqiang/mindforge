## SPIKE_PLAN §7 Go / No-go

**结论：Go.**

理由：

1. R1–R4 全部 Go。无 Conditional 项。
2. balanced-2000 在「mount / edit-sync / pan」三个维度均位于 Go 阈值之内。
3. 与早些时间的同日 baseline `2026-05-15.md`（memo outline 改完后的 Go 版本）对比，
   edit-sync 在 2000 节点上从 49.65 ms 降到 39.69 ms（≈20% 降低），pan p95 从
   9.2 ms 降到 8.8 ms（噪声内）。P0+P1+P2 没有引入回归。
4. `core/` / `layout/` / `render/` / `outline/` 边界保持完整：
   - `core/` 不依赖 React / Tiptap。
   - `layout/` 不 touching DOM。
   - `render/` 不直接 import `applyDocOp`（通过 props 拿 surface callback）。
   - `outline/` 行级 memo，行内通过 `NodeEditorSlot` 共用 editor 框架。
5. 模块公开 API 同步到 [CORE_API.zh-CN.md](../CORE_API.zh-CN.md)、
   [STATE_MODEL.zh-CN.md](../STATE_MODEL.zh-CN.md)；ADR-0002 / ADR-0004
   的 mitigations 状态已从「承诺」改为「已实现」。

## 进入 v0.1-release 之前推荐补一发 hardening

不强制（不是 Conditional），但值得在第一个 release iteration 早期处理：

1. **slice 订阅迁移**：把 `App` 的 `useSyncExternalStore(subscribe, getDoc)`
   改成只订阅 `rootId` + 节点切片；OutlinePane / SpikeCanvas 改成 `subscribeNode`
   风格的 slice hook。spike 阶段允许整 doc 订阅（[STATE_MODEL §8](../STATE_MODEL.zh-CN.md#8-store-公开访问)）。
2. **集中式 `EditorBridgeState`**：目前散在 `NodeEditor.tsx` ref/state 里；
   当「AI 流式 patch」或「多 surface 同时编辑」上线时再统一。
3. **outline 虚拟化**：超过 2000 行时 React 的 reconciler 会成为瓶颈，
   提前接入 react-window 之类的库，但暂时还没到。
4. **键盘快捷键 + 折叠 affordance**：节点 collapsed op 已经能用，但 UI 没有按钮 / hotkey。
